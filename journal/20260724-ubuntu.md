# Installing Kubernetes on Ubuntu

## Goal
Is to understand the difference between Ubuntu and Fedora for the installation of Kubernetes.

## What I learned
- There is a difference between Fedora and Ubuntu that impacts Kubernetes;

## Progress
Installed Kubernetes in the same way as on Fedora yesterday. However this resulted in a working cluster.
So there are key differences between Ubuntu and Fedora. I think it has to with firewall or security settings.

## Different repo used on Fedora
I used the default Fedora repo for installing Kubernetes. This is a difference from Ubuntu, in which I used the k8s repo.
I will try this same method for Fedora.
```bash
# 1. Add Kubernetes official repo (v1.32 example - change if needed)
cat <<EOF | sudo tee /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=https://pkgs.k8s.io/core:/stable:/v1.32/rpm/
enabled=1
gpgcheck=1
gpgkey=https://pkgs.k8s.io/core:/stable:/v1.32/rpm/repodata/repomd.xml.key
EOF

# 2. Install packages
sudo dnf install -y kubelet kubeadm kubectl

# 3. Hold the packages (like apt-mark hold)
sudo dnf versionlock add kubelet kubeadm kubectl
```
